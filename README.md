# Locks 1.20.1 (多平台锁模组)

基于 Architectury 框架开发的 Minecraft 1.20.1 多平台（Forge/Fabric）锁与安全机制模组。

## 📥 简介 (Introduction)
Locks 是一个专注于在 Minecraft 中实现物理容器锁以及相关玩法的模组。它不仅能够保护你的私有财产，还能与世界生成的遗迹进行深度结合，为你探索地牢、村庄等结构时带来更具沉浸感的开锁与寻宝体验。

## ✨ 核心特性 (Features)

*   **多层级锁具**: 提供木锁、铁锁、钢锁、金锁、钻石锁、下界合金锁等。锁具的品质将直接影响其开锁难度与抗破坏能力。
*   **钥匙系统与钥匙环**: 自由复制、配对你的钥匙。使用**钥匙环**将钥匙串联管理，或是使用更高阶的**万能钥匙**。
*   **撬锁小游戏**: 忘带钥匙？使用**开锁器**体验自定义撬锁机制（带有动态动画效果与音效反馈），不同材质锁芯的挑战难度截然不同。
*   **暴力破锁**: 允许使用斧头等高伤害工具强行破锁！系统将动态计算工具材质效率、附魔（如耐久减免）等属性。强行破坏某些高级锁具甚至会触发特定的反弹伤害（如电击惩罚）。
*   **世界生成与战利品池整合**: 完美融入原版世界。在世界生成的战利品宝箱（如废弃矿井、下界要塞、村庄铁匠铺等）上自动随机生成符合群系难度的物理锁。
*   **跨平台兼容**: 采用 Architectury 架构原生构建，共享核心逻辑的同时无缝支持 Fabric 与 Forge API。

## 🛠️ 构建指南 (Build Instructions)

本项目采用 Gradle 进行构建。开发环境已配置完整的多模块系统。

### 前置要求
*   Java 17
*   Minecraft 1.20.1

### 编译与打包
在项目根目录下打开终端，运行：
```bash
# Windows
gradlew build

# Linux / macOS
./gradlew build
```
编译成功后，可以直接到对应平台目录获取 Mod 文件：
*   Forge 产物路径: `forge/build/libs/`
*   Fabric 产物路径: `fabric/build/libs/`

## ⚙️ 配置系统 (Configuration)

配置分为两部分：
1. **配置文件 (Config)**：位于 `config/` 目录下。例如 `locks-server.toml` 可精细控制工具破锁耗损、生成概率、附魔惩罚等机制。(修改配置后会同步应用)
2. **数据包 (Data Pack)**：锁的生成规则、权重以及白名单等已高度数据化。修改或覆盖 `data/locks/loot_tables/` 下的表单能够自由调整自然生成的结构锁。

## 📝 开发者说明
在构建和维护本项目代码时，严格遵循 SOLID 设计原则以及模块化分离的要求。尽量保证单个类的单一职责，并在跨平台处理时多使用通用接口。
